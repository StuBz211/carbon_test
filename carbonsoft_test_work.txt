Тестовое задание на позицию разработчика python (full-stack, middle)

Требуемые навыки:
  - python
  - flask, jinja2
  - бд, orm
  - linux, bash
  - js
  - css
  - git

Необходимо разработать систему сбора информации.
Система должна состоять из 2-х компонентов: сервер на базе flask, клиент-демон для linux.
Клиент периодически отправляет данные на сервер. Сервер их сохраняет и отображает пользователю.

Оценка времени на выполнение задания 4-8 часов.

Требование к клиенту-демону:
 - работает на linux    
 - параметром задаём ип-порт сервера, дефолт - 127.0.0.1:8001
 - каждые 10 секунд отправляет на сервер текущую утилизацию cpu в %
 - можно писать полностью на bash. Крепкое знание bash приветствуется

Требования к серверу:
 - на базе flask (flask для проекта vps, django для проекта биллинг) желательно
 - БД - любая sql, лучше sqlite
 - БД может быть временная(:memory:), без сохранения на диск
 - для работы с БД желательно использовать orm, но допускается dbapi2
 - сервер получает данные от клиента через rest-протокол, сохраняет в БД
 - страница / для пользователя содержит:
   - таблицу со 100 последними записями
   - таблица с возможностью сортировки(JS)
   - отдельно агрегированные данные:
     - min/max/avg 100 последних записей
     - min/max/avg всех записей в БД
   - для оформления страницы желательно использовать стили
   - если данные будут динамически обновляться через ajax - отлично, но можно рендерить и на jinja

Требования к оформлению задания:
 - результат в виде python пакета
 - результат должен быть размещён на публичном git ресурсе(github, bitbucket, https://gitlab_test.carbonsoft.ru/)
 - развёртывание pip3 install <prj_repo_url>
 - в README написать как запустить сервер и клиент-демон
 - работа без root привелегий
 - отдельный функционал из требований может отсутствовать
